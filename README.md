# PisoWise 🇵🇭💰

> **AI-powered financial literacy platform for low-income Filipino families**
> Built with React + Vite · Firebase Spark · Groq Llama 3.3 70B · Capacitor Android

---

## ✨ Features

### 💸 Income & Expense Tracking
- Log transactions via **text input**
- 16 expense + 12 income categories with Filipino context
- Filter by type, date range, and category — plus free-text **search** across description/note/category
- Attach a **receipt photo** to any transaction (compressed client-side, stored directly on the transaction doc — no Firebase Storage/Blaze plan needed)
- **Recurring bills & income** — set up anything that repeats weekly/monthly/yearly, get a due-soon/overdue reminder on Dashboard and Transactions, and log it with one tap when paid
- **Export to CSV** for any filtered view, Excel/Sheets-ready
- Real-time 7-day spending chart

### 🤖 AI Budget Generator (Groq Llama 3.3 70B)
- Personalized monthly budgets based on income, family size, and goals
- Filipino context: OFW remittances, paluwagan, sari-sari stores, 13th month pay
- Pie chart breakdown of current spending
- Save and revisit monthly budgets

### 🐷 Savings Goals
- Create goals with custom colors, nicknames, and deadlines
- Track progress with animated visual bars
- AI-generated savings strategies per goal
- Quick-add funds feature
- **Savings Challenges** — pre-built plans (52-Week Challenge, ₱1,000/Buwan, 30-Day No-Spend, ₱20/Araw) that set up a goal with a suggested target, deadline, and schedule in one tap

### ⛓️ Debt Manager
- Track all debts: bank loans, 5-6, Pag-IBIG, SSS, credit cards, family loans
- **Automatic 5-6/bombay loan warning** with escape strategies
- AI debt repayment plan (narrative advice from Groq)
- **Deterministic Payoff Calculator** — snowball vs. avalanche simulation using each debt's actual interest rate and minimum payment, showing months-to-debt-free, total interest, and how much avalanche saves over snowball
- Quick payment logging with progress tracking

### 💳 Card Wallet (Security-First)
- Add only **last 4 digits** — full card numbers, CVV, and expiry dates are **never collected**
- **EmailJS OTP verification** required before adding or editing any card
- 10+ Philippine bank card types (BDO, BPI, GCash, Maya, Metrobank, etc.)
- 10 custom gradient designs with live card preview
- Visual credit/debit card display with chip and contactless icon

### 📊 Financial Health Score (0–100)
- Calculated from savings rate, emergency fund, debt-to-income ratio
- AI explanation of your score and top improvement action
- Real-time score updates as you log data

### 🤖 AI Insights & Coaching
- **Weekly spending analysis** with actionable recommendations
- **Financial Coach Chat** — conversational AI powered by Llama 3.3 70B
- Quick prompts for common Filipino financial questions
- Health score explanations with personalized advice

### 📚 Financial Literacy Lessons
- 5 beginner-to-intermediate lessons in Filipino
- Topics: Budgeting, Savings, Debt, Emergency Fund, Investing
- Interactive quizzes with pass/fail tracking
- **AI Q&A** — ask questions about each lesson in real-time

### 🔔 Push Notifications (FCM)
- Budget alerts at 80% and 100% of monthly income
- Weekly spending summaries (every Monday)
- Debt payment reminders (3 days before due)
- Monthly financial health check reminders

## 🔐 Security Architecture

### Card Wallet Security
- **Never stored:** Full card numbers, CVV, expiry dates
- **Only stored:** Last 4 digits + card type + nickname + color scheme + `verified: true`
- **Firestore rules** actively reject documents containing `fullNumber`, `cvv`, or `expiryDate` fields
- **EmailJS OTP** required before any card is added or edited
- OTP tokens expire after 10 minutes and are single-use (deleted after verification)

### Authentication
- Firebase Authentication (Email/Password)
- Auth state persisted via Firebase SDK
- Route guards protect all private pages
- Firestore rules verify `request.auth.uid` matches document `userId` on every read/write

### Data Isolation
- All Firestore queries include `userId` field matching
- Security rules double-check ownership on every operation
- No cross-user data access possible

## 📊 Firestore Usage (Spark Plan Optimization)

### Collections
| Collection    | Reads/Write Pattern          | Limit per day |
|---------------|------------------------------|---------------|
| `users`       | 1 read on login, rare writes | Low           |
| `transactions`| Batch load (limit 60)        | Medium        |
| `savings`     | All goals at once            | Low           |
| `debts`       | All debts at once            | Low           |
| `cards`       | All cards at once            | Low           |
| `insights`    | Last 5 only                  | Low           |
| `budgets`     | Current month only           | Low           |
| `recurringTransactions` | All active bills at once (small list) | Low |
| `otpTokens`   | Create + delete per OTP      | Very Low      |

### Optimizations
- Zustand store caches all data in memory — no repeated Firestore reads
- `transactionsLoaded`, `savingsLoaded` etc. flags prevent duplicate fetches
- Transactions limited to 60 most recent
- Insights limited to 5 most recent
- Budgets only fetch current month
- OTP tokens are checked for expiry client-side on every verify attempt, so expired codes never work even without a cleanup job (Spark plan has no Cloud Functions to run one)

## 🏆 Financial Health Score Algorithm

```
Score (0–100) = Sum of:
  Savings Rate (0–30pts):
    ≥20% saved → 30pts | ≥10% → 20pts | ≥5% → 10pts

  Emergency Fund (0–25pts):
    ≥6 months expenses → 25pts | ≥3 months → 18pts | ≥1 month → 10pts

  Debt-to-Income (0–25pts):
    No debt → 25pts | <10% → 22pts | <25% → 15pts | <50% → 8pts

  Active Savings Goals (0–10pts):
    ≥3 goals → 10pts | ≥1 goal → 6pts

  Expense Ratio (0–10pts):
    ≤50% of income → 10pts | ≤70% → 7pts | ≤85% → 4pts | ≤100% → 1pt
```
